Accessibility

All color combinations must meet WCAG 2.1 contrast ratio requirements.

Contrast Requirements

LevelRatioUse Case
AA Normal4.5:1Body text, labels
AA Large3:1Headings (18px+ or 14px+ bold)
AAA Normal7:1Enhanced accessibility

Best Practices

  • Use semantic tokens first. Only reach for raw palette values when building custom components that need specific shade control.
  • Test in both themes. Always verify color choices render well in both light and dark mode.
  • Don't rely on color alone. Pair color indicators with icons, text, or patterns for users with color vision deficiencies.
  • Maintain consistent meaning. Red for errors, amber for warnings, green for success — keep these associations stable.
  • Check contrast ratios. Use browser dev tools or accessibility checkers to verify WCAG compliance.
  • Use opacity for layering. Prefer bg-background/80 over separate color definitions for overlays.
