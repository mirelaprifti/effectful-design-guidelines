Elevation Levels

Elevation levels define how "high" an element appears above the base surface. Higher levels demand more attention and indicate interactive overlays.

Design rationale: Use the minimum elevation needed to establish hierarchy. Most page content should sit at Level 0 or Level 1, reserving higher levels for overlays and modals.

Surface

Level 0 – Flat

shadow-none + border

Page background, inline content, table rows. No shadow; uses border for separation.

Level 1 – Raised

shadow-sm

Cards, sticky headers, nav bars. Subtle lift to distinguish from the background.

Level 2 – Overlay

shadow-md

Dropdowns, popovers, tooltips. Clearly floats above the surface.

Level 3 – Modal

shadow-xl / shadow-2xl

Modals, dialogs, command palettes. Highest elevation, paired with a backdrop overlay.

Dark Mode Considerations

Shadows are less visible on dark backgrounds. In dark mode, rely more on surface color differences and borders to establish hierarchy.

Design rationale: On dark surfaces, shadows blend into the background. Instead, elevate dark UI by using lighter surface colors for higher levels. Combine subtle shadows with surface tints for the best result.

Light Mode

Shadows are the primary elevation cue. Use the full shadow scale freely.

Level 0: border only
Level 1: shadow-sm
Level 2: shadow-md

Dark Mode

Combine lighter surfaces with subtle shadows. Higher = lighter surface.

Level 0: base surface
Level 1: lighter + shadow
Level 2: even lighter + shadow

Best Practices

✓

Use Minimum Elevation

Most UI should be Level 0 or 1. Reserve Level 2-3 for overlays and modals that need to dominate focus.

✓

Pair Shadow with Radius

Larger shadows pair with larger border-radius values. See Border Radius + Shadows for pairings.

✓

Combine Shadow + Border

For cards and containers at Level 0-1, use shadow-sm with a border for subtle, defined edges.

✓

Avoid Stacking Shadows

Don't layer elevated elements on top of each other. A dropdown inside a modal should use the modal's surface, not add its own shadow.
