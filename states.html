State Overview

Every interactive element should communicate its current state clearly. Consistent state patterns reduce cognitive load and build user confidence.

State Trigger Visual Cue Purpose
DefaultNoneBase appearanceResting state of the element
HoverMouse enterBrightness shift, bg tintIndicates interactivity
ActiveMouse down / tapScale down, darker shadeConfirms the press is registered
FocusTab / keyboard navFocus ring (outline)Keyboard navigation indicator
DisabledProgrammaticReduced opacity, no cursorElement is unavailable
LoadingAsync actionSpinner, skeleton, progressOperation is in progress
ErrorValidation failureRed border, error messageInput needs correction
SuccessValidation passGreen border, check iconInput is valid

Best Practices

✓

Always Provide Focus Rings

Never remove focus outlines without replacing them. Use focus-visible to limit rings to keyboard navigation only.

✓

Don't Rely on Color Alone

Pair color changes with text, icons, or shape changes. This ensures states are perceivable for color-blind users.

✓

Use 150ms Transitions

Animate all state changes with transition-all duration-150. Instant changes feel jarring; too-slow transitions feel sluggish.

✓

Explain Disabled States

When possible, use a tooltip or helper text to explain why an element is disabled and what the user can do to enable it.
